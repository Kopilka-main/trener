# Тренер

Веб-приложение для персонального тренера: ведение клиентов, база упражнений и тренировок, проведение тренировки с таймером отдыха и фиксацией результатов.

Реализован MVP тренера по 10 эскизам из корня репозитория. Мобильный UI (макет 390×844), оптимизирован для просмотра в мобильном Chrome или DevTools (iPhone 14 Pro 393×852).

## Стек

- **Backend**: Node.js + Express + better-sqlite3 + Zod (TypeScript)
- **Frontend**: React 18 + Vite + TypeScript + TanStack Query + Tailwind CSS v4 + @dnd-kit + lucide-react

## Установка и запуск

Требуется Node.js ≥ 20.

```powershell
# из корня проекта
npm install
npm --prefix server install
npm --prefix web install

# запуск дев-сервера и фронтенда одной командой
npm run dev
```

После запуска:
- Бэкенд: `http://localhost:3001` (REST API под `/api/...`).
- Фронтенд: `http://localhost:5173` (Vite проксирует `/api/*` на бэкенд).

При первом старте автоматически создаётся `server/data.db` и наполняется тестовыми данными (10 клиентов, ~20 упражнений, 5 шаблонов тренировок) на основе эскизов.

Чтобы пересоздать данные с нуля — удалите `server/data.db` и перезапустите.

## Полезные команды

| Команда | Что делает |
|---|---|
| `npm run dev` | Поднимает сервер и фронтенд параллельно |
| `npm run build` | Собирает оба проекта в production |
| `npm --prefix server run seed` | Сидирует БД (если пустая) |

## Сценарий проверки

В режиме мобильного устройства (DevTools → Toggle device toolbar → iPhone 14 Pro):

1. Открыть `http://localhost:5173`.
2. На вкладке «Клиенты» удерживать на «Алина Кузнецова» 0.5 с — открывается превью (эскиз 05).
3. «Редактировать» → изменить заметку → «Сохранить».
4. Тап по карточке → «Тренировки» → видно «Низ · Гипертрофия» в готовых.
5. «Выбрать из базы» → пик «Низ · Гипертрофия» → появляется «Текущая тренировка».
6. «Начать тренировку» (эскиз 07) → тап на любой подход → ввести факт → «Готово» (запускается таймер отдыха со звуком).
7. «Завершить» → итоги (эскиз 08) → заметка + RPE 7 → «Сохранить в историю» → тренировка перешла в историю.

## Что не вошло в MVP

- Клиентский вход (роли клиента) — следующий этап.
- Загрузка фото клиентов и медиа упражнений.
- Push-уведомления, аналитика прогрессов, синхронизация между устройствами.
