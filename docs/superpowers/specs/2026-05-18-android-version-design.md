# Android-версия приложения «Тренер» — дизайн

Дата: 2026-05-18

## Цель

Сделать Android-приложение «Тренер» для раздачи тренерам напрямую (APK, без
Google Play). Приложение работает офлайн, данные хранятся локально на телефоне.
В Android-версии остаётся только тренерская часть — клиентское приложение и
выбор роли не нужны.

## Вводные (решения пользователя)

- **Распространение:** APK раздаётся тренерам напрямую, без публикации в Google Play.
- **Данные:** хранятся локально на устройстве каждого тренера, офлайн, без сервера.
  Без синхронизации между устройствами.
- **Стартовые данные:** база упражнений и шаблоны тренировок есть, список клиентов
  пустой — тренер заводит своих.
- **Только тренер:** клиентская часть и экран выбора роли удаляются.
- **Без логина:** экраны входа и регистрации удаляются, приложение открывается
  сразу на рабочем экране.

## Архитектура

### Отдельный проект

Создаётся новая папка в корне репозитория — `trener-mobile/`. Это самостоятельный
проект, не связанный с `github_demo`: своя `package.json`, свой `node_modules`,
своя сборка. Веб-демо (`github_demo`) остаётся нетронутой.

Внутрь копируется фронтенд из `github_demo` и сразу вычищается от всего
клиентского — остаётся чистое тренерское приложение. Затем добавляется Capacitor,
который собирает APK из статической сборки Vite.

```
trener-mobile/
├── src/                  # только тренерский код, клиентского нет
├── dist/                 # сборка Vite
├── android/              # нативный Android-проект (Capacitor)
├── capacitor.config.ts
├── vite.config.ts
└── package.json
```

### Технология упаковки — Capacitor

Capacitor берёт собранную статику из `dist/`, кладёт её в нативный Android-проект
(`android/`) и собирает APK. Внутри APK — WebView с тем же React-приложением,
данные в `localStorage`. Сервер не нужен, всё работает офлайн.

Вариант Capacitor выбран как минимально-затратный: переиспользуется уже
работающая бессерверная сборка `github_demo` (весь бэкенд там переписан на
`localStorage` в `demo-backend.ts`). Альтернативы отклонены:
- **PWA + TWA (Bubblewrap)** — требует хостинг сайта и интернет, плохо ложится на
  офлайн-приложение, заточен под Play Store.
- **Нативная переписка (React Native / Kotlin)** — огромный объём работы,
  выбрасывает рабочий код, избыточно.

## Изменения в коде приложения

Берётся копия `github_demo/src` и приводится к тренерскому-только виду.

### Удаляются файлы целиком

- `pages/RoleSelectPage.tsx` — выбор роли больше не нужен.
- `pages/LoginPage.tsx`, `pages/RegisterPage.tsx` — вход убран.
- `pages/ClientProfilePage.tsx` — экран только клиентской части.
- Клиентские стартовые данные в `api/demo-seed.ts`.

### Правятся файлы

- `App.tsx` — убирается вся логика ролей и авторизации (`app_role`, `trener_auth`,
  `RoleSelectPage`, `isClient`, клиентская ветка `Routes`, маршруты
  `/trainer/login` и `/trainer/register`). Остаётся только тренерский `Routes`.
  Стартовый экран — `/trainer/clients`.
- `lib/routes.ts` — удаляются `CLIENT_BASE`, `DEMO_CLIENT_ID` и клиентские ветки в
  `appBase`, `clientWorkoutsPath`, `backTarget`. Функции упрощаются под одну роль.
- `components/BottomTabBar.tsx` — убирается параметр `role`, остаётся тренерский
  таб-бар.
- `api/demo-backend.ts` — выкидывается клиентская логика, если она там есть.
- `api/demo-seed.ts` — упражнения и шаблоны тренировок остаются, массив клиентов
  и связанные занятия/история — пустые.

### Общие экраны

`ClientsPage`, `ClientEditPage`, `ClientWorkoutsPage`, `KnowledgeBasePage`,
`ExerciseEditorPage`, `WorkoutBuilderPage`, `ActiveWorkoutPage`,
`WorkoutSummaryPage`, `CalendarPage`, `TrainerPage`, `TrainerEditPage` —
используются тренерской частью, остаются как есть.

## Сборка APK

### Capacitor

Добавляются зависимости `@capacitor/core`, `@capacitor/cli`, `@capacitor/android`.

`capacitor.config.ts`:
- `appId`: `com.trener.mobile`
- `appName`: «Тренер»
- `webDir`: `dist`

### Правки сборки

В `vite.config.ts` уже стоит `base: './'`, роутер — `HashRouter`. Это именно то,
что нужно Capacitor (приложение грузится с `file://`), менять почти ничего не
придётся.

### Иконка и сплэш-экран

Генерируются утилитой `@capacitor/assets` из одной картинки-логотипа (PNG). Если
готового логотипа нет — делается простая заглушка.

### Поток сборки

```
npm run build      → Vite собирает dist/
npx cap sync       → копирует dist/ в android/
сборка APK         → Gradle (assembleDebug) или Android Studio
```

### Тип APK

Для раздачи тренерам напрямую достаточно debug-APK — он ставится с телефона при
включённой «установке из неизвестных источников», подпись keystore не нужна.
Релизный подписанный APK — опционально на будущее.

### Прерёквизит окружения

Для сборки APK на машине разработчика нужны JDK 17 и Android SDK (обычно через
Android Studio). Без них проект можно довести до состояния «готов к сборке», но
сам APK собрать не получится. Наличие проверяется на этапе реализации.

## Хранение данных и риски

- Данные (клиенты, тренировки, история) — в `localStorage` WebView. Переживают
  перезапуск приложения и его обновление.
- Теряются при удалении приложения или «Очистить данные» в настройках Android.
- Для v1 этого достаточно. Экспорт/импорт данных в файл — отдельная задача на
  будущее, в текущий объём не входит.

## Тестирование

- Собрать APK, установить на реальное устройство или эмулятор.
- Пройти тренерский сценарий: добавить клиента → создать тренировку → провести с
  таймером отдыха → завершить → проверить историю.
- Убедиться: нет экрана выбора роли, нет логина, клиентские экраны недостижимы.
- Проверить, что данные сохраняются после перезапуска приложения.

## Что не входит в объём (YAGNI)

- Публикация в Google Play, релизная подпись keystore.
- Синхронизация и облачное резервное копирование.
- Экспорт/импорт данных.
- Изменения в `github_demo` (веб-демо) и в основном приложении `web` + `server`.
