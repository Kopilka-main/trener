# Тренер — статичная демо-версия

Самодостаточная сборка приложения «Тренер» для публикации на **GitHub Pages**.
Серверная часть (Express + SQLite) здесь не нужна: весь бэкенд повторён в браузере.

## Чем отличается от основного приложения (`../web` + `../server`)

| | Основное приложение | Эта демо-версия |
|---|---|---|
| Бэкенд | Express + SQLite | `src/api/demo-backend.ts` — роутер поверх `localStorage` |
| Данные | общая база на сервере | свои в каждом браузере |
| Роутер | `BrowserRouter` (`/trainer/...`) | `HashRouter` (`/#/trainer/...`) |
| Хостинг | нужен Node-сервер | любая статика, в т.ч. GitHub Pages |

Стартовые данные (клиенты, упражнения, шаблоны, занятия, профиль тренера)
лежат в `src/api/demo-seed.ts` и загружаются в `localStorage` при первом открытии.

> ⚠️ Данные демо живут только в одном браузере. Очистка данных сайта
> возвращает приложение к стартовому состоянию.

## Локальный запуск

```bash
npm install
npm run dev      # http://127.0.0.1:5173
```

## Сборка

```bash
npm run build    # результат в dist/
npm run preview  # локальный просмотр собранной версии
```

## Публикация на GitHub Pages

В репозитории есть workflow `.github/workflows/deploy-demo.yml`. После пуша
в `main`/`master`:

1. Открыть **Settings → Pages** репозитория.
2. В разделе **Build and deployment** выбрать **Source: GitHub Actions**.
3. Workflow соберёт `github_demo/` и опубликует сайт.

Сайт будет доступен по адресу `https://<логин>.github.io/<репозиторий>/`.
Префикс `base: './'` в `vite.config.ts` делает сборку независимой от имени
репозитория — менять конфиг не нужно.
